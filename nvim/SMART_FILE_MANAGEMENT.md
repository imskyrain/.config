# 智能文件管理功能说明

## 🎯 新增的智能行为

为了更接近传统编辑器（如 Sublime Text、VSCode）的体验，已优化文件的新建、保存和关闭行为。

---

## 📋 功能详解

### 1. 新建文件 (Ctrl+N)

**当前行为**:
```
Ctrl+N → 创建空白 buffer → 可以编辑 → 保存时提示命名
```

#### 使用示例

```vim
" 按 Ctrl+N
" 创建空白 buffer，可以立即开始编辑

" 编辑完成后按 Ctrl+S
" 弹出：保存为: _
" 输入文件名：test.js
" 保存成功
```

#### 推荐工作流

**方法1：先编辑后命名**
```vim
Ctrl+N           " 新建空白buffer
输入内容
Ctrl+S           " 保存时提示命名
输入：test.js
```

**方法2：使用 Oil 创建（推荐用于需要特定位置的文件）**
```vim
<leader>e        " 打开 Oil
导航到目标目录
o                " 新建行
输入：test.js
:w               " 保存（创建文件）
回车             " 打开文件
```

**方法3：直接用命令创建**
```vim
:e path/to/file.js    " 直接编辑（会创建文件）
```

---

### 2. 智能保存 (Ctrl+S)

**传统 Vim 行为**:
```
未命名文件 → Ctrl+S → 报错：No file name
```

**新的智能行为**:
```
未命名文件 → Ctrl+S → 提示输入文件名 → 保存
已命名文件 → Ctrl+S → 直接保存
```

#### 使用示例

```vim
" 场景 1：意外创建了未命名 buffer
:enew
输入一些内容
按 Ctrl+S
" 弹出：保存为: _
" 输入文件名即可保存

" 场景 2：正常的已命名文件
按 Ctrl+S
" 直接保存，无提示
```

#### 优点

✅ 不会因为忘记命名而无法保存
✅ Ctrl+S 在任何情况下都能工作
✅ 避免数据丢失

---

### 3. 智能关闭 (Ctrl+W)

**传统 Vim 行为**:
```
未保存的文件 → Ctrl+W → 报错提示 → 需要手动处理
```

**新的智能行为**:
```
未命名且有修改 → Ctrl+W → 弹出选择：保存/不保存/取消
已命名且有修改 → Ctrl+W → Vim 默认提示
无修改         → Ctrl+W → 直接关闭
```

#### 使用示例

```vim
" 场景 1：未命名文件有内容
:enew
输入一些内容
按 Ctrl+W
" 弹出选择：
" 1. 保存     → 提示输入文件名
" 2. 不保存   → 直接关闭，丢弃内容
" 3. 取消     → 继续编辑

" 场景 2：已命名但未保存
编辑 test.js
按 Ctrl+W
" Vim 提示：No write since last change (add ! to override)
" 可以 :w 保存或 :bd! 强制关闭

" 场景 3：已保存的文件
按 Ctrl+W
" 直接关闭
```

#### 优点

✅ 防止意外丢失未保存的内容
✅ 清晰的三选一界面，不会误操作
✅ 保护重要的工作成果

---

### 4. 退出前检查 (:q, :qa, :wqa)

**新增的保护机制**:
```
未命名且有内容的文件 → 退出 → 阻止并提示
```

#### 使用示例

```vim
" 场景 1：试图退出未保存的未命名文件
:enew
输入一些重要内容
:q
" ❌ 被阻止，显示通知：
" "未命名文件有未保存的内容，请先保存（Ctrl+S）或使用 :q! 强制退出"

" 解决方法：
Ctrl+S         " 保存文件
或
:q!            " 强制退出（会丢失内容）

" 场景 2：空的未命名文件
:enew
" 不输入任何内容
:q
" ✅ 直接退出（没有内容，无需保存）

" 场景 3：已命名的文件
:e test.js
编辑内容
:q
" Vim 默认行为：提示未保存
```

#### 优点

✅ 最后一道防线，防止意外丢失内容
✅ 只检查真正有内容的未命名文件
✅ 空文件可以直接退出，不会打扰

## 🔄 完整工作流程示例

### 工作流 1：快速创建新文件

```
方法 1 - 先编辑后命名（适合临时想法）：
1. 按 Ctrl+N
2. 开始编辑
3. 按 Ctrl+S
4. 输入：src/components/Header.tsx
5. 按 Ctrl+W 关闭

方法 2 - 直接指定文件名（适合已知文件路径）：
1. :e src/components/Header.tsx
2. 开始编辑
3. 按 Ctrl+S 保存
4. 按 Ctrl+W 关闭

方法 3 - 使用 Oil 创建（适合浏览式创建）：
1. <leader>e
2. 导航到 src/components/
3. o（新建行）
4. 输入：Header.tsx
5. :w（保存创建）
6. 回车打开
```

### 工作流 2：从未命名文件保存

```
1. 用其他方式创建了未命名 buffer（如 :enew）
2. 写了一些内容
3. 按 Ctrl+S
4. 输入文件名：notes.md
5. 保存成功
```

### 工作流 3：安全关闭

```
1. 编辑了未命名文件
2. 忘记保存，直接按 Ctrl+W
3. 弹出选择，选择"保存"
4. 输入文件名
5. 保存并关闭
```

### 工作流 4：防止意外退出

```
1. 在未命名文件中写了很多代码
2. 不小心按了 :qa
3. 被阻止，显示警告
4. 意识到需要保存
5. 按 Ctrl+S 保存
6. 再次 :qa 成功退出
```

## 📖 快捷键总结

| 快捷键 | 功能 | 智能行为 |
|--------|------|----------|
| `Ctrl+N` | 新建文件 | 立即提示输入文件名 |
| `Ctrl+S` | 保存文件 | 未命名文件提示输入文件名 |
| `Ctrl+W` | 关闭文件 | 未保存的未命名文件弹出选择 |
| `:q` / `:qa` | 退出 | 未命名有内容的文件被阻止 |
| `:q!` | 强制退出 | 绕过所有检查（会丢失内容）|

## 💡 使用技巧

### 1. 快速创建多个文件

```vim
" 传统方式：
:e src/utils/helpers.js
:e src/utils/constants.js
:e src/utils/api.js

" 新方式：
Ctrl+N → 输入 src/utils/helpers.js
Ctrl+N → 输入 src/utils/constants.js
Ctrl+N → 输入 src/utils/api.js
```

### 2. 创建嵌套目录的文件

```vim
" Neovim 会自动创建不存在的目录
Ctrl+N
输入：src/components/auth/LoginForm.tsx
" 即使 src/components/auth/ 不存在也会自动创建
```

### 3. 临时记录想法

```vim
" 如果只是临时记录，不想保存
:enew         " 创建临时 buffer
写一些内容
" 使用完后直接关闭：
:bd!          " 强制关闭，不保存
```

### 4. 使用 Oil 创建文件（推荐）

```vim
" 方法 1：使用 Ctrl+N（需要知道完整路径）
Ctrl+N → 输入完整路径

" 方法 2：使用 Oil 浏览器（更直观）
<leader>e      " 打开 Oil
浏览到目标目录
o              " 新建行
输入文件名
:w             " 保存（创建文件）
回车打开文件
```

## ⚠️ 注意事项

### 1. 强制退出会丢失内容

```vim
:q!    " 强制退出，忽略所有警告
:qa!   " 强制退出所有，忽略所有警告
```

**警告**: 使用 `!` 会绕过所有保护机制，未保存的内容会丢失！

### 2. 某些命令不会触发检查

```vim
:bd!          " 强制删除 buffer，不会提示
:bw!          " 强制擦除 buffer，不会提示
ZQ            " Vim 快捷退出（等同于 :q!）
```

### 3. 批量操作

```vim
" 关闭所有未命名文件（会逐个提示）
:bufdo bd

" 强制关闭所有（不会提示，会丢失内容）
:bufdo bd!
```

## 🔧 自定义配置

如果你不喜欢某些行为，可以修改配置：

### 禁用 Ctrl+N 的文件名提示

编辑 `lua/core/keymap.lua`:

```lua
-- 改回简单的新建 buffer
map("n", "<C-n>", "<cmd>enew<CR>", { desc = "新建文件" })
```

### 禁用退出前的检查

编辑 `lua/core/autocmd.lua`，删除或注释掉 `QuitPre` 的 autocmd。

### 调整保存行为

编辑 `lua/core/keymap.lua`，修改 `<C-s>` 的映射。

## 🆚 与传统编辑器对比

### Sublime Text / VSCode

| 功能 | Sublime/VSCode | 优化后的 Neovim | 原始 Neovim |
|------|----------------|----------------|-------------|
| 新建文件 | Ctrl+N → Untitled-1 | Ctrl+N → 提示命名 | :enew |

| 退出未保存 | 警告提示 | 警告并阻止 | :q 报错 |

### 优势

✅ 更安全：多重保护防止数据丢失
✅ 更友好：清晰的提示和选择
✅ 更高效：快捷键始终可用
✅ 保留 Vim 特性：仍然可以用 `:enew`, `:w` 等命令

## 🎯 总结

### 核心改进

1. **Ctrl+N** - 新建文件时立即命名
2. **Ctrl+S** - 智能保存，自动处理未命名文件
3. **Ctrl+W** - 智能关闭，防止丢失内容
4. **退出检查** - 阻止未保存的未命名文件退出

### 设计理念

- **安全第一**: 防止意外丢失工作成果
- **用户友好**: 清晰的提示和选择
- **保持灵活**: 仍然可以用 `!` 强制执行
- **渐进增强**: 不影响传统 Vim 命令
